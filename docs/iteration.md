# 迭代过程

## V1 项目骨架

建立 React/FastAPI 前后端分离结构，加入 Docker Compose、README、环境变量模板。

## V2 论文发现与排序

接入 arXiv 实时检索，实现论文元信息抽取和综合排序解释。

## V3 PDF 精读与 RAG

实现 PDF 文本解析、chunk 切分、sentence-transformers embedding 和基于证据的问答。

## V4 增强分析

加入结构化总结、Reviewer 视角分析、研究趋势分析、论文关系图谱和自动文献综述。

## V5 交互轨迹与提交整理

在系统界面展示 Agent 执行轨迹，并预留 `codex_sessions/` 用于提交完整交互轨迹。

## V6 关注领域追踪与 PDF 任务反馈

新增关注领域列表，使用 localStorage 保存关注方向、已见论文和上次刷新结果；打开页面或手动刷新时自动从 arXiv 获取最新论文并标记 `NEW`。在线 PDF 解析改为后台任务，前端展示下载、解析文本、切分 chunk、构建向量索引、完成或失败状态。

## V7 顶会收件箱与阅读体验优化

将关注模块改为“顶会论文收件箱 + 分析工作台”：重复添加同一关注领域时更新原卡片，首次刷新作为 baseline，后续刷新才显示 `NEW`。默认使用 Semantic Scholar 检索顶会论文，arXiv 作为预印本探索和 hybrid 补充。结构化总结和 Reviewer 分析改为流式 LLM 任务，前端支持 Markdown 渲染和源码切换。

## V8 收件箱语义修正与等待反馈

移除左侧临时检索入口，所有论文发现统一通过收件箱完成。追踪表单改为关注名称、研究关键词、论文来源、会议范围等科研语义字段，并在表单中解释当前查询范围。后端修正顶会模式 fallback：`顶会论文` 只返回 Semantic Scholar 结果，限流或超时会在对应卡片明确失败；只有 `顶会 + arXiv` 才允许 arXiv 补充。补齐收件箱刷新、排序、趋势、图谱、综述和问答的局部等待反馈。

## V9 Semantic Scholar 申请期稳定化

新增 Semantic Scholar API Key 健康状态；未配置 key 时前端默认使用 `arXiv 预印本`，确保系统仍可稳定完成最新论文追踪、自动获取元信息和工作台分析。删除收件箱中的自定义会议栏，会议范围改为完整会议组下拉；拿到 key 后写入 `.env` 并重启服务即可默认恢复 `顶会论文` 检索。

## V10 问答证据、关系分析与筛选增强

页面标题改为“论文追踪与分析工作台”。问答模块明确说明答案由 Kimi/OpenAI-compatible API 基于本地 PDF 证据生成，下方依据片段支持页码、相似度和展开原文。论文关系图谱改为“论文关系分析”，展示摘要语义相似的论文对和共享关键词。会议范围新增“全部会议”，收件箱新增最新优先、引用量优先和综合推荐三种返回排序方式。
